Providers - Request a statement of achievement

User journey for Providers's to request a statement of achievement for learners who have not completed their T Level.

From: R&C Team

Published: 25 January 2022

Last updated: 25 January 2022

Journey overview

AO's inform ESFA of students results for each registered component.

Users add a result against components (Core and or Specialism(s)) for registered assessment entries.

For example if a student has taken an exam for their core in the summer of 2021, the AO must be able to update our service with the grade that the learner achieved for that exam.

End users

  • Learner records editor (user role in the service)
  • Administrators

Pre requisites before requesting a statement of achievement

  • Provider must have added organisations address (where the certificates should be sent).
  • Learner must have been initially registered by an AO.
  • Learner must be in a withdrawn state - provider must inform AO who will place learner into withdrawn state via the AO R&C service.
  • Learner must not have had a SoA requested in the last 3 weeks (can only request one after 3 weeks of requesting first statement).
  • Learner must have completed an industry placement and or received a core component result.
  • Provider must create a T Level record for the student before requesting a statement of achievement.

Current Functionality

Request statement of achievement on individual basis.

Sceanrios catered for:

  • Learner not registered by exam board.
  • No postal address information in system for provider.
  • Printing not available yet (Too early to request SoA).
  • Learner not recorded as a withdrawn status in the system.
  • Statement already requested
  • Provider has not created a T level record for the learner in the system
  • Provider has not updated learners T level record with information about Industry placement status and or has not received a core component result
