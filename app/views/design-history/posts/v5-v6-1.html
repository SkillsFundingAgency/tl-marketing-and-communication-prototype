Version 5 to version 6-1

A list of all changes made from July 2022 - November 2022

Site-wide changes

  • Added a "Known issues" dropdown to version 5. This shows details of all technical issues and limitations currently present in that version

    • Due to some issues having a disproportionate burden it was decided that we would leave them in and mention them in a dropdown menu.

  • Added a start page to the prototype

    • In order to showcase what the start page of this product would look like, we made a dummy one that will be implemented in full on the live version of the service

  • Added an accessibility statement

    • We added an information page in the footer detailing all the accessibility checks and any issues that we are aware of but unable to fix

  • Fixed validation for a number of different inputs across the page

    • Some of the inputs had incorrect validation, causing them to act in ways unintended if a user was to input incorrect information. This has been rectified and all inputs should have correct validation

  • Altered breadcrumbs in various pages

    • Page design changed, and it was decided we would alter the way we display breadcrumbs on pages

  • Cleaned up code for various pages, making it more legible

    • Altered the code in a way that it didn't affect content, however allowed developers to better read the code. Any future changes that may need to be made should be easier to do

Providers dashboard

  • Various content changes

    • A number of changes were made to the content of the tiles in the dashboard

  • Updated tile design

    • The design of the tiles needed to be changes in version 6, leading to the current design present today (as of version 6-1)

  • Added new tiles

    • As we added new journeys and services we also needed to add new tiles to access them

AO dashboard

  • Updated tile design

    • Like with the providers dashboard, the AO dashboard also needed to be updated to a new design

Manage learner records

  • Added a search learner records page that filters learners based on what link the user follows

    • Added a page that displays the various start years that learners may have joined in. Following one of these links will filter the learners so that only the ones that started at the chosen year is displayed

  • Added ability to search for a learner via ULN

    • On the same page that asks you to filter via year we also have a search bar for if a user would prefer to search for an individual learner

  • Added ability to filter learners

    • After selecting a year to filter by we also added the ability to further filter uses via checkboxes on the sidebar. Users can filter by subject, or the state of the learners course

  • Added ability to sort learners by last name

    • Users can also sort learners by A-Z or Z-A by clicking on the correct link. This sorting process should only target the learners last name.

  • Added ability to search for learners based on last name or ULN

    • after sorting via year, if a user wanted to search for a specific learner we also added a search bar where users can type in learner details in order to find them.

  • Added dynamic pagination and dynamic text that changes based on the amount of learners present, or what page the user is on

    • Instead of displaying all learners at once, we have limited displayed learners down to 10 per page and added the ability to cycle between pages. There will some text stating how many learners there are total which will change based on the amount

  • Added and updated data present in prototype

    • It was decided that in order to correctly test the prototype we needed some real data, so we altered the learner data files and inputted more learners

  • Updated design of T Level record page

    • Once a learner is selected it will take you to their specific page. The design of this page has changed numerous times over the different versions to fit user needs

  • Made maths, English, and IP statuses able to be changed and made them dynamic across learners

    • Users can now change details in the individual learner pages, these changes are saved on said learners and can be accessed again without the details being overwritten

  • Added ability to request a replacement document and added the journey to do so

    • Should a learner lose their documents providers can request that a replacement be sent out to them. This feature only because available when learners have completed their course

  • Simplified IP journey

    • Removed and condensed multiple pages in the IP journey to create a less confusing and more succinct experience

  • Multiple changes to IP journey to suit shifting design requirements
  • Added two new options to IP journey: Learner needs more time and Learner will not complete their industry placement

    • Options added for if a learner will never complete their IP journey but is still continuing with the course, as well as if a learner is planning on completing their IP but haven't done so yet

  • Added a variety of success banners based on learner IP status

    • Depending on what the user selects a different banner will be displayed, depicting the status of the learners industry placement

  • Added a tag next to the learners name, displaying whether or not their record has been completed

    • These tags were previously removed, but then added back as it was decided that it would help users know at a glance the status of their learners

Download T Level results

  • Created Download T Level results page

    • Added a page where users can download the results for all active learners who are in the final year of their T Level

Bulk upload

  • Created bulk upload journey

    • Created a bulk upload journey consisting of 3 different pages. Users can upload multiple CSV files of their learners and these learners will be uploaded to the system. There is a validation method in place that will inform users if they are using the wrong type of file or data format.
